The methodology

Agentic architecture —
the principles I work by.

These aren't abstract ideas. They're the specific decisions I made while rebuilding Steller — and the ones I help other founders make.

01
Optimise for the context window, not the service boundary
Architecture decisions must first answer: can an AI hold the entire relevant flow in one context window? If not, the design is wrong — regardless of its engineering merit.
Context visibility over distribution
02
Keep the golden flow in one traceable path
The sequence that moves revenue must be readable top to bottom without crossing a service boundary. If an AI can't trace it in one sitting, redesign it.
One flow · One file · No invisible breaks
03
Write AGENTS.md before you write any code
The file that tells an agent where to start is worth more than any single code file. It cuts session orientation from 20 minutes to 2. Written once, paid forward forever.
Docs compound like interest
04
Make the wrong move require explicit confirmation
Guardrails, confirmation gates, protected systems. An agent that cannot accidentally destroy a production system is worth ten agents that might.
Safety is architecture, not comment

A decade of closing deals in markets defined by risk, information scarcity, and complex internal politics.
